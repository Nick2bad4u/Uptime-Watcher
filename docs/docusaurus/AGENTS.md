# Agents Guide For Docusaurus

## Scope

This file applies to `docs/docusaurus/**`, the Docusaurus documentation
workspace for Uptime Watcher.

## Project Shape

- Treat this directory as a nested npm workspace with its own `package.json`,
  TypeScript config, Docusaurus config, and package lock.
- Root scripts usually orchestrate the full docs pipeline. Prefer root commands
  such as `npm run docs:typedoc`, `npm run docusaurus:build`,
  `npm run check:docusaurus`, and `npm run build:eslint-inspector` when
  verifying integration with the main repository.
- Workspace scripts are appropriate for site-only work, for example
  `npm run --workspace docs/docusaurus typecheck` or running commands from
  `docs/docusaurus`.

## Docusaurus Content And Config

- Keep `docusaurus.config.ts`, `sidebars.ts`, route base paths, deployment
  paths, and GitHub Pages settings aligned with
  `https://nick2bad4u.github.io/Uptime-Watcher/`.
- Use MDX deliberately. Prefer plain Markdown unless React components are
  required for the page.
- Keep custom React components under `src/` typed and accessible. Match the
  existing Docusaurus and React patterns.
- Keep CSS modules and generated CSS type files in sync. Run the workspace CSS
  type check when editing `*.module.css`.
- Do not change global theme behavior, PWA config, search config, or plugin
  config without a docs build.

## Generated Outputs

- TypeDoc currently writes generated API documentation into
  `docs/docusaurus/docs`. Do not manually patch generated API pages; update
  source TSDoc, `typedoc.config.json`, `typedoc.local.config.json`, or the
  TypeDoc plugins.
- `docs/docusaurus/static/eslint-inspector` is generated by
  `npm run build:eslint-inspector`.
- `docs/docusaurus/static/storybook` is generated by
  `npm run build:docs-storybook`.
- `docs/docusaurus/build` and `.docusaurus` are build outputs. Do not edit them
  as source.

## TypeDoc And Sidebars

- If a sidebar entry breaks after API generation, regenerate TypeDoc and update
  `sidebars.ts` to match real generated document IDs.
- Do not create fake generated pages to satisfy stale sidebar IDs.
- Keep TypeDoc plugin behavior in `typedoc-plugins/` covered by tests under the
  root tooling test suite when logic changes.
- Local TypeDoc output paths in `typedoc.local.config.json` are developer
  specific. Do not copy them into the default config.

## Verification

- For config, sidebars, MDX, or route changes, run
  `npm run docusaurus:build` from the repository root.
- For TypeScript or component changes, run `npm run check:docusaurus`.
- For generated API docs, run `npm run docs:typedoc` before the Docusaurus
  build.
- For CSS module edits, run
  `npm run --workspace docs/docusaurus css:types:check`.
- For links and frontmatter, run `npm run docs:check` and
  `npm run docs:check-links`.
